# Скрапинг сотрудников РГПУ им. А. И. Герцена

## Цель

Получить данные сотрудников со страниц `https://atlas.herzen.spb.ru/teachers?page=1` - `https://atlas.herzen.spb.ru/teachers?page=54` и сформировать CSV-файл с полями:

- `name` - ФИО сотрудника;
- `email` - почта;
- `phone` - телефон;
- `link` - ссылка на профиль.

## Структура

- `PythonSoup/soup.py` - реализация на Python с использованием `requests` и `BeautifulSoup`.
- `PythonSoup/pythonsoup.csv` - результат работы Python-скрипта.
- `Puppeteer/index.js` - реализация на Node.js с использованием Puppeteer.
- `Puppeteer/puppeteer.csv` - результат работы Puppeteer-скрипта.
- `.gitignore` - исключает зависимости, виртуальные окружения, кеши и системные файлы.

## Принцип работы

1. Скрипт проходит по страницам каталога преподавателей с 1 по 54.
2. На каждой странице собирает ФИО и ссылку на профиль преподавателя.
3. Для каждого профиля открывает отдельную страницу и ищет контактные данные.
4. Если почта или телефон отсутствуют, записывается значение `Не указано`.
5. Собранные данные сохраняются в CSV с разделителем `;`.

## BeautifulSoup

Python-реализация использует обычные HTTP-запросы через `requests`. HTML разбирается библиотекой `BeautifulSoup`; нужные элементы выбираются по CSS-классам, найденным в разметке сайта.

Запуск:

```bash
cd PythonSoup
python soup.py
```

## Puppeteer

Puppeteer-реализация открывает страницы в браузере Chromium и извлекает данные через DOM API внутри страницы. Такой подход полезен для сайтов, где часть данных может зависеть от работы браузера.

Установка зависимостей:

```bash
cd Puppeteer
npm install
```

Запуск:

```bash
npm start
```

## Результат

Задание выполнено двумя способами. В обеих реализациях данные собираются из общего списка преподавателей, затем дополняются почтой и телефоном со страниц профилей и сохраняются в CSV-файлы.
