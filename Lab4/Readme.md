# Лабораторная работа 4 — Size2JSON

## ✅ Описание решения

В рамках работы реализовано небольшое веб-приложение, которое по POST-запросу на маршруте `/size2json` принимает файл из multipart/form-data (поле `image`) и возвращает JSON-строку вида:

```json
{"width": 123, "height": 456}
```

Если передан невалидный/неподдерживаемый файл, возвращается:

```json
{"result": "invalid filetype"}
```

Маршрут `/login` возвращает JSON с логином (MOODLE):

```json
{"author": "1155288"}
```

## 🚀 Как запустить (локально)

### 1) FastAPI (Python)

1. Убедитесь, что активировали виртуальное окружение (или используйте свой):
   - Windows: `python -m venv .venv` / `.venv\Scripts\activate`
2. Установите зависимости:
   ```bash
   pip install fastapi uvicorn pillow
   ```
3. Запустите сервер:
   ```bash
   uvicorn python.app:app --reload --port 8000
   ```

После запуска приложение доступно по адресу:

- **http://localhost:8000/**  (форма, маршрут `/`)
- **http://localhost:8000/login**
- **http://localhost:8000/size2json**

> ⚠️ В текущей конфигурации приложение работает по **HTTP**, а не по HTTPS. Для HTTPS надо запустить Uvicorn с `--ssl-keyfile` / `--ssl-certfile` или использовать прокси/терминатор (ngrok, Caddy и т.п.).

### 2) Node.js (Express)

1. Перейдите в каталог `node.js`
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите сервер:
   ```bash
   node app.js
   ```

После запуска приложение доступно по адресу:

- **http://localhost:1234/**  (форма, маршрут `/`)
- **http://localhost:1234/login**
- **http://localhost:1234/size2json**

## 📁 Структура проекта

- `python/app.py` — FastAPI-реализация
- `python/form.html` — HTML-форма для загрузки изображения (отправляется на порт 8000)
- `node.js/app.js` — Express-реализация
- `node.js/form.html` — HTML-форма для загрузки изображения (отправляется на порт 1234)

## 🧪 Тестирование

Вы можете проверить работу `/size2json` через curl:

```bash
curl -F "image=@./test.png" http://localhost:8000/size2json
```

## 📝 Ограничения и замечания

- 📌 **Формат:** работает только с изображениями, которые могут корректно открыть Pillow (FastAPI) или Sharp (Node.js).
- 🚫 **Проверка типа:** если файл не является изображением, возвращается JSON `{"result":"invalid filetype"}`.
- 🐞 **Node.js:** реализация содержит ошибку в обработке загруженного файла (используется `req.file.file` вместо `req.file.buffer`), поэтому в текущем виде она не будет возвращать корректные размеры.
- 🔧 **Дополнительно (не реализовано):**
  - асинхронная отправка формы через fetch/ajax без перезагрузки страницы
  - сохранение состояния (какой последний файл загружен и результаты)
  - вывод thumbnail загруженного изображения на страницу
  - работа через HTTPS без дополнительных настроек
